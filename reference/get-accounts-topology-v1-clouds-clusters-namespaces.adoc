= 클라우드의 네임스페이스 나열
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/clouds/{cloud_id}/clusters/{cluster_id}/namespaces`#

컬렉션에서 일치하는 모든 리소스를 JSON 배열로 반환합니다. 요청에 개별 필드가 지정된 경우, items JSON 배열에는 지정된 순서대로 각 일치하는 리소스에 대해 요청된 필드가 포함됩니다.



== 매개변수

[cols="5*"]
|===
| 이름 | 유형 | 에서 | 필수 요소입니다 | 설명 


| account_id | 문자열 | 경로 | 참  a| 
포함하는 계정 리소스의 ID

* 예: {{.Account}}




| cloud_id | 문자열 | 경로 | 참  a| 
포함된 클라우드 리소스의 ID



| 클러스터_ID | 문자열 | 경로 | 참  a| 
포함된 클러스터 리소스의 ID



| 포함 | 문자열 | 쿼리 | 거짓  a| 
컬렉션 목록을 표시할 때 어떤 필드를 반환해야 하는지 지정합니다.

URL 예: "include=id", "include=id,name"

OpenAPI 예: "id"

* 예시: id,name,kubernetesLabels




| 제한 | 문자열 | 쿼리 | 거짓  a| 
컬렉션 목록을 표시할 때 반환할 최대 리소스 수를 나타냅니다.

URL 예: "limit=2"

OpenAPI 예제: "2"



| 필터 | 문자열 | 쿼리 | 거짓  a| 
컬렉션 목록을 표시할 때 리소스를 반환하기 위해 어떤 필드를 일치시켜야 하는지를 나타냅니다.

지원되는 연산자는 다음과 같습니다.

* eq - 같음
* lt/gt - 미만/초과
* lte/gte - 이하/이상


URL 예: "filter=field%20eq%20%27value%27"

OpenAPI 예: "field eq 'value'"

|===

* 최소 길이: 1
* 최대 길이: 255




| namespaceState | 문자열 | 참  a| 
네임스페이스의 상태를 나타내는 값을 포함하는 JSON 문자열입니다. 정의된 값은 다음과 같습니다.

* "removed" - 네임스페이스가 클러스터에서 삭제되었거나 클러스터가 관리되지 않지만 네임스페이스를 참조하는 리소스가 있을 수 있습니다




| namespaceStateDetails | 배열[link:#type_astra_stateDetail["type_astra_stateDetail"]] | 참  a| 
"state"가 "state a", "state b" 또는 "state c"로 전환된 이유에 대한 추가 세부 정보를 나열하는 JSON 객체의 JSON 배열입니다. 정보가 없는 경우 이 배열은 비어 있습니다. 정의된 값은 다음과 같습니다.

* 최소 항목: 0




| kubernetesLabels | 배열[link:#type_astra_label["type_astra_label"]] | 거짓  a| 
네임스페이스와 연결된 Kubernetes 레이블을 각각 포함하는 JSON 객체의 JSON 배열입니다. 정의된 값은 다음과 같습니다.

* Kubernetes 레이블 배열 스키마를 준수합니다.




| clusterID | 문자열 | 참  a| 
네임스페이스가 포함된 클러스터의 전역적으로 고유한 식별자를 포함하는 JSON 문자열입니다. 정의된 값은 다음과 같습니다.

* Astra 식별자 스키마를 준수합니다.




| systemType | 문자열 | 거짓  a| 
네임스페이스가 시스템 네임스페이스 유형임을 나타내는 JSON 문자열입니다. 이 필드는 네임스페이스가 어떤 유형의 시스템 네임스페이스인지 식별합니다(버전 1.1에만 해당). 정의된 값은 다음과 같습니다.

* "kubernetes" - 일반 Kubernetes(K8s) 시스템 네임스페이스(예: "kube-system")
* "netapp" - NetApp 시스템 네임스페이스(예: "trident" 또는 "netapp-monitoring")
* "openshift" - OpenShift 시스템 네임스페이스(예: "openshift-monitoring")
* "rke" - Rancher 네임스페이스(예: "cattle-logging")
* "other" - 다른 곳에 반드시 맞지 않는 일반적인 시스템 네임스페이스(예: "ingress-nginx", "tekton-system")
