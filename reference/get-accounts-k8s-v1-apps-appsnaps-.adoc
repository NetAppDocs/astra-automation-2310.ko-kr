= 애플리케이션 스냅샷을 검색합니다
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/k8s/v1/apps/{app_id}/appSnaps/{appSnap_id}`#

JSON 응답 본문에 저장된 Application Snapshot API 리소스를 반환합니다.



== 매개변수

[cols="5*"]
|===
| 이름 | 유형 | 에서 | 필수 요소입니다 | 설명 


| account_id | 문자열 | 경로 | 참  a| 
포함하는 계정 리소스의 ID

* 예: {{.Account}}




| app_id | 문자열 | 경로 | 참  a| 
포함하는 앱 리소스의 ID



| appSnap_id | 문자열 | 경로 | 참  a| 
목록에 표시할 appSnap 컬렉션의 ID

|===


== 응답

[source]
----
Status: 200, Returns the stored Application Snapshot API resource in the JSON response body.
----
[cols="4*"]
|===
| 이름 | 유형 | 필수 요소입니다 | 설명 


| 유형 | 문자열 | 참  a| 
리소스의 미디어 유형입니다. 정의된 값은 다음과 같습니다.

* "application/astra-appSnap"




| 버전 | 문자열 | 참  a| 
리소스 버전입니다. 정의된 값은 다음과 같습니다.

* "1.0"
* "1.1"
* "1.2"




| ID입니다 | 문자열 | 참  a| 
리소스의 전역 고유 식별자입니다. 정의된 값은 다음과 같습니다.

* UUIDv4 스키마를 준수합니다.




| 이름 | 문자열 | 참  a| 
사용자가 지정한 스냅샷 이름을 포함하는 JSON 문자열입니다. 이름을 지정하지 않거나 예약된 일정에 따라 스냅샷이 시작될 경우 시스템에서 스냅샷 이름을 자동으로 할당합니다. 정의된 값은 다음과 같습니다.

* 최소 길이: 1
* 최대 길이: 63
* DNS 1123 레이블 스키마를 준수합니다.




| scheduleID | 문자열 | 거짓  a| 
스냅샷이 스케줄에 의해 트리거되었음을 나타내는 선택적 스케줄 ID와 스냅샷을 트리거한 스케줄을 지정하는 JSON 문자열입니다. 스케줄에 의해 시작된 스냅샷에만 존재합니다. 정의된 값은 다음과 같습니다.

* Astra 식별자 스키마를 준수합니다.




| snapshotAppAsset | 문자열 | 거짓  a| 
스냅샷에 해당하는 애플리케이션 자산의 ID가 포함된 JSON 문자열입니다. 이 필드는 애플리케이션 스냅샷 상태가 "completed"일 때 반환됩니다. 정의된 값은 다음과 같습니다.

* Astra 식별자 스키마를 준수합니다.




| 상태 | 문자열 | 참  a| 
스냅샷의 작동 상태를 나타내는 값을 포함하는 JSON 문자열입니다. 정의된 값은 다음과 같습니다.

* "pending" - 애플리케이션 스냅샷 검색이 예약되었습니다.
* "discovering" - 애플리케이션 스냅샷을 검색하는 중입니다
* "running" - 애플리케이션 스냅샷이 정상적으로 실행 중입니다.
* "completed" - 애플리케이션 스냅샷이 완료되었습니다.
* "failed" - 애플리케이션 스냅샷이 영구적인 실패 상태입니다
* "removed" - 현재 애플리케이션 스냅샷에 접근할 수 없습니다.
* "unknown" - 애플리케이션 스냅샷 상태를 확인할 수 없습니다.




| stateUnready | 배열[string] | 참  a| 
JSON 문자열의 JSON 배열로, 각 문자열은 애플리케이션 스냅샷이 "running" 상태로 전환할 수 없는 이유 또는 "failed" 상태로 전환된 이유를 나타냅니다. 정의된 값은 다음과 같습니다.

* 최소 길이: 1
* 최대 길이: 127




| hookState | 문자열 | 거짓  a| 
스냅샷의 일부로 실행 중인 실행 후크의 상태를 나타내는 값을 포함하는 JSON 문자열입니다. 정의된 값은 다음과 같습니다.

* "failed" - 하나 이상의 실행 후크가 실패했습니다.
* "success" - 적용 가능한 후크가 0개 이상 있었고 모두 성공적이었습니다.




| hookStateDetails | 배열[link:#type_astra_stateDetail["type_astra_stateDetail"]] | 거짓  a| 
"hookState"가 "failed"로 전환된 이유에 대한 추가 정보를 나열하는 JSON 객체의 JSON 배열입니다. 정보가 없는 경우 이 배열은 비어 있습니다.

* 최소 항목: 0




| 메타데이터 | link:#type_astra_metadata["type_astra_메타데이터"] | 참  a| 
리소스와 연결된 클라이언트 및 서비스 지정 메타데이터입니다. 정의된 값은 다음과 같습니다.

* Astra 메타데이터 스키마를 준수합니다. 생성 시 지정하지 않으면 레이블이 없는 메타데이터 객체가 생성됩니다.
